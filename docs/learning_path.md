# 量化学习路线

这份路线对应当前项目结构。建议你每次只推进一个小实验，保留实验记录。

## 第 1 阶段：建立直觉

核心问题：

- 收益率、波动率、回撤是什么？
- 为什么不能随机切分时间序列？
- 为什么训练集表现好不代表策略能赚钱？

练习：

- 跑通 `python -m quant_for_fun.examples.train_baseline`
- 修改 synthetic 数据的漂移率和波动率，观察指标变化
- 把 `probability_threshold` 从 0.50 改到 0.60、0.65，对比交易次数和收益

## 第 2 阶段：特征工程

核心问题：

- 动量、反转、波动率、成交量因子各自表达什么？
- 特征是否用了未来数据？
- 标签 horizon 对模型难度有什么影响？

练习：

- 增加 RSI、布林带宽度、过去 N 日最大回撤等特征
- 分别预测 1 日、5 日、20 日 forward return
- 把二分类标签改成三分类：上涨、震荡、下跌

## 第 3 阶段：模型训练

核心问题：

- 分类模型和回归模型分别适合什么策略表达？
- 模型概率如何转换为仓位？
- walk-forward validation 为什么比单次切分更可靠？

练习：

- 尝试 Logistic Regression、Gradient Boosting、Random Forest
- 增加 walk-forward 训练函数
- 记录每次实验的参数、指标和结论

## 第 4 阶段：回测严谨性

核心问题：

- 交易成本、滑点、换手率如何影响结果？
- 信号生成时间和下单成交时间是否一致？
- 最大回撤比单纯收益更重要的场景是什么？

练习：

- 增加手续费和滑点假设
- 增加仓位上限，例如 0%、50%、100%
- 输出 yearly return 和 monthly return 表格

## 第 5 阶段：真实数据

建议先从免费且容易下载的数据开始，例如：

- CSV 文件：你手动下载后放到 `data/raw/`
- Yahoo Finance：之后可用 `yfinance` 接入
- 加密货币日线：交易所导出的 CSV

练习：

- 新建 `quant_for_fun/data/csv_loader.py`
- 统一字段：`open`, `high`, `low`, `close`, `volume`
- 对真实标的重复 baseline 实验
